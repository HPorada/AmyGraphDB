Visualisation¶

Functions to visualise query results¶

Functions used to visualise query results and other JSON files which have a structure compatible with ArangoDB. One of the function uses Graphviz library, the other combines NetworkX to create a graph and Pyvis to display it.

visualisation_functions.graphviz_graph(filename, input_dir='./management/json_data', output_dir='./management/graphviz', int_questions=False, sequences=False, engine='dot', direction='LR')¶

This method visualises a chosen JSON file using Graphviz library and saves the result in chosen directory.

Parameters

  • filename – (str) Name of the JSON file which is to be visualised.

  • input_dir – (str) Path to the directory with the chosen file.

  • output_dir – (str) Path to the directory where graph is to be saved.

  • int_questions – (boolean) Default: False - connections with question nodes are not shown in the graph.

  • sequences – (boolean) Default: False - labels of sequence nodes are names or “seq:[number]”, not full sequences.

  • engine – (str) Default: ‘dot’. Graph layout engine. Other: ‘neato’, ‘twopi’, ‘circo’, ‘fdp’, ‘osage’, ‘patchwork’, ‘sfdp’.

  • direction – (str) Default: ‘LR’. Direction of the graph (left -> right). Other: ‘RL’, ‘TB’, ‘BT’.

visualisation_functions.networkx_graph(filename, input_dir='./management/json_data', output_dir='./management/networkx', int_questions=False, sequences=True, general_remarks=True)¶

This method visualises a chosen JSON file using NetworkX and Pyvis libraries and saves the result in chosen directory.

Parameters

  • filename – (str) Name of the JSON file which is to be visualised.

  • input_dir – (str) Path to the directory with the chosen file.

  • output_dir – (str) Path to the directory where graph is to be saved.

  • int_questions – (boolean) Default: False - connections with question nodes are not shown in the graph.

  • sequences – (boolean) Default: True - full sequences are shown when sequence nodes are being hovered over.

  • general_remarks – (boolean) Default: True - attributes ‘general remarks’ are shown when interaction nodes are being hovered over.
