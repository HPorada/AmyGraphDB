Database management¶

Combined query functions¶

Combined query functions execute proper query function based on database structure passed as an argument.

queries.contains_fragment(structure, database, fragment, filename='result', directory=None)¶

This method executes a query filtering the database of chosen structure in search of sequences containing chosen fragment.

Parameters

  • structure – (str) Structure of the chosen database (“simple”, “extended” or “extendedv2”).

  • database – (StandardDatabase) Database in which query is to be executed.

  • fragment – (str) Sequence fragment which is to be looked for.

  • filename – (str) Name of the file where query result is to be saved. Optional.

  • directory – (str) Path to the directory where file with query result is to be saved. Optional.

queries.custom_query(database, query, filename='result', directory=None)¶

This method allows to execute a custom query in chosen database and save it with chosen name in chosen directory.

Parameters

  • database – (StandardDatabase) Database in which query is to be executed.

  • query – (str) AQL query that is to be executed.

  • filename – (str) Name of the file where query result is to be saved. Optional.

  • directory – (str) Path to the directory where file with query result is to be saved. Optional.

queries.filter_questions(structure, database, q1=None, q2=None, q3=None, filename='result', directory=None)¶

This method executes a simple query filtering the database of chosen structure based on answers to 3 questions:

1. Is the interactor affecting interactee’s aggregating speed? (Faster aggregation/Slower aggregation/No aggregation/No effect/No information)

2. Do fibrils of the interactee elongate by attaching to monomers/oligomers/fibrils of the interactor? (Yes, direct evidence/Yes, implied by kinetics/No/Formation of fibrils by the interactee is inhibited/No information)

3. Is interaction resulting in heterogeneous fibrils consisting of interactor and interactee molecules? (Yes/No/No information)

Parameters

  • structure – (str) Structure of the chosen database (“simple”, “extended” or “extendedv2”).

  • database – (StandardDatabase) Database in which query is to be executed.

  • q1 – (str) Answer to the first question defining the interaction. Optional.

  • q2 – (str) Answer to the second question defining the interaction. Optional.

  • q3 – (str) Answer to the third question defining the interaction. Optional.

  • filename – (str) Name of the file where query result is to be saved. Optional.

  • directory – (str) Path to the directory where file with query result is to be saved. Optional.
